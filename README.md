# Diagnóstico de Problemas em Computadores

Guia de troubleshooting para identificação e resolução de problemas comuns em computadores.  
Projeto criado para estudo de processos de suporte técnico e atendimento ao usuário.

## Computador não liga

Possíveis verificações:

1. Verificar se o cabo de energia está conectado corretamente
2. Testar a tomada com outro dispositivo
3. Verificar se a fonte de alimentação está funcionando
4. Testar os módulos de memória RAM
5. Verificar possíveis sinais da placa-mãe (LED ou beep)

## Computador lento

Possíveis causas:

1. Alto consumo de CPU ou memória
2. Muitos programas iniciando com o sistema
3. Falta de espaço em disco
4. Arquivos temporários acumulados
5. Presença de malware

Soluções possíveis:

- Verificar uso de recursos no Gerenciador de Tarefas
- Desativar programas desnecessários na inicialização
- Limpar arquivos temporários
- Executar verificação com antivírus

## Problemas de conexão com a internet

Procedimentos de diagnóstico:

1. Verificar conexão física do cabo de rede
2. Testar conexão Wi-Fi
3. Reiniciar modem ou roteador
4. Verificar endereço IP
5. Executar teste de conectividade (ping)

## Objetivo do projeto

Este projeto tem como objetivo praticar:

- Diagnóstico de problemas de computadores
- Organização de processos de suporte técnico
- Documentação de soluções comuns em ambientes corporativos
